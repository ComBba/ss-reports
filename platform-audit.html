GAP 1선언 ✅ vs 실제 라이브의 큰 간극

사내 SERVICE-INVENTORY.md는 121개 중 95개 "✅ Used"로 표기하지만(게다가 D53 이전 stale 모델 포함), HONEST-SCOPE.md 기준 demonstrated-live는 7행뿐. 심사자가 인벤토리를 "라이브"로 오독하지 않도록 HONEST-SCOPE를 항상 우선 노출해야 함.

GAP 2Govern/Optimize = 관리형 제품이 아닌 "자체구현"

Registry·Anomaly·Policy·Security·Compliance·Simulation은 GCP 관리형 제품이 아니라 우리 에이전트/코드. "플랫폼 제품을 사용"이 아니라 "개념을 자체구현"으로 정직히 표현해야 함(Built-With 태그가 제품 사용으로 오인될 위험).

GAP 3Agent Runtime(관리형) 미사용 — Cloud Run 대체

serve.py 타이틀이 "ADK Agent Runtime"이지만 실체는 FastAPI on Cloud Run. SERVICE-INV "all 22 agents hosted here"는 계획. Track 2 핵심 차별점이라면 실제 Agent Runtime 배포 1건이 필요.

GAP 4암호·집행 계층 스텁 (AP2 / Model Armor 플릿 / mTLS)

AP2 실서명·WebAuthn 검증 미구현(NotImplementedError), Model Armor는 A2A ingress만(플릿 미적용), Agent Gateway mTLS 선언-미집행. 보안 "집행"이 데모에서 부분적.

GAP 5완전 부재 컴포넌트

A2UI · UCP · RAG/Vertex AI Search · Agent Sandbox · Managed Training = 제품코드 0건. 일부는 SERVICE-INV에 ✅로 선언되어 있어 정정 필요.
